Getting Started

How it works

  • Preparation: a user interacts with a privacy-preserving smart contract through a secure communication channel to negotiate the parameters;
  • Issuing: an issuer blindly issues a credential to the user;
  • Verifying: the user shows the issued credential and the verifier confirms the credential.
  • Tracing: the corresponding entities such as issuer and verifier invokes the blockchain bearing an auditable transaction to reveal the blind.

After obtaining these parameters, the user can fill out and submit the information.

Tracing

Two phases are divided in this section: one is using the user's identity $\xi^u$ to trace credential $\zeta_1$, the other is using credential $\zeta_1$ to trace identity $\xi^u$.


Each operation will be recorded by invoking the smart contract.

The smart contract returns the results corresponding to the value you given.
